# Data Structures and Algorithms (DSA)

Welcome to my repository for the Data Structures and Algorithms course. This space serves as a portfolio of my problem-solving journey, algorithmic thinking, and practical implementations using C++.

---

## 📖 Course Overview
This course focuses on the design, analysis, and implementation of fundamental data structures and algorithms. Through rigorous lab sessions and intensive assignments, I explored how data organization directly impacts program execution time, memory management, and overall system efficiency. 

Key themes include abstract data types (ADTs), memory optimization, pointer manipulation, and foundational algorithms for searching, sorting, and simulation.

---

## 📂 Repository Structure & Project Breakdown

### 🔹 Assignments
*   **Assignment 1 (`/Assignment1`)**: Focused on custom structured data using `Student.h` to process student records and export corresponding academic results (`Marks.txt`).
*   **Assignment 2 (`/Assignment2`)**: An advanced implementation utilizing a custom List Abstract Data Type (`List.h`) to handle pointer-based operations dynamically for a structured dataset.

### 🔹 Lab Exercises & Mini-Projects
*   **Lab 1 (`/Lab1`)**: Groundwork on basic memory allocation and custom structures combining `Lab1.cpp` and `Student.h`.
*   **Labs 2, 3, & 4 (`Lab2.cpp`, `Lab3.cpp`, `Lab4.cpp`)**: Sequential progression through core concepts such as Linked Lists, Stacks, Queues, and basic Trees.
*   **Flight Simulation (`flight_simulation.cpp`)**: A real-world application mapping out complex logic and dynamic event processing using advanced data structure coordination.

## 💡 Course Reflection & Personal Key Takeaways

### 🧠 Intellectual Growth
Before this course, writing code was mostly about getting the correct output. Now, I understand that *how* data is stored is just as critical. Designing custom classes like `List.h` taught me to visualize how memory is structurally mapped out under the hood, and how easily pointer leaks or dangling references can break a system.

### 🛑 Biggest Challenge Overcome
The transition from static arrays to dynamic, linked nodes (especially during Assignment 2) was incredibly challenging. Debugging segmentation faults and tracing pointers manually on paper became a mandatory part of my workflow. Overcoming this gave me a massive boost in mental tracking and debugging patience.

### 🌟 Most Rewarding Project
Building the **Flight Simulation (`flight_simulation.cpp`)**. Seeing abstract concepts translate into a working model that mimics real-life queue systems and scheduling behaviors made all the frustrating debugging sessions completely worth it. It proved to me how DSA forms the silent backbone of complex real-world software.
